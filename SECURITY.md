# Security Policy

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues.

Instead, please report them through one of the following channels:

- **GitHub Private Reporting** — use the [Security tab](../../security/advisories/new) on this repository to privately report a vulnerability
- **Email** — send details to the repository maintainer

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### For Upstream Projects

If the vulnerability is in NemoClaw or OpenShell itself (not specific to this cookbook), please report it through NVIDIA's official channels:

- [NVIDIA Vulnerability Submission Form](https://www.nvidia.com/object/submit-security-vulnerability.html)
- Email: psirt@nvidia.com (PGP key available at [nvidia.com/security/pgp-key](https://www.nvidia.com/en-us/security/pgp-key))

For more information, see [NVIDIA's Product Security](https://www.nvidia.com/en-us/security) page.
